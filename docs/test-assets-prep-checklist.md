# 测试资源准备清单

后续真实 OAuth POC 只允许操作用户明确准备的测试资源。本阶段不操作任何真实网盘文件。

## 用户需要准备什么

建议准备一个小测试目录：

```text
panjie-smoke-source/
  hello.txt
  note.txt
  sample-image.png
```

文件要求：

- 文件必须是用户自己创建或有权测试的内容。
- 不放隐私资料。
- 不放大文件。
- 不放版权争议资源。
- 文件名尽量简单，便于验证重命名和移动。

## 测试目录授权

后续 POC 需要用户明确确认是否允许创建：

```text
盘姬测试/panjie-smoke-YYYYMMDD-HHMMSS
```

规则：

- 只操作测试目录。
- 不删除任何文件。
- 不覆盖任何原文件。
- 不操作私人资料目录。
- 测试失败时只记录脱敏原因。

## 测试分享链接准备

后续真实转存测试需要用户准备一个自有测试分享链接和提取码。

要求：

- 不把完整链接发到聊天。
- 不把提取码发到聊天。
- 可以在本机 UI 或 `.env.local.test` 中填写。
- 日志和文档中必须显示为：

```text
shareUrl: <redacted>
extractCode: <redacted>
```

## 是否允许创建测试分享

用户需要确认：

- 是否允许软件对测试目录创建分享链接。
- 是否允许设置有效期。
- 是否允许生成提取码。

未确认前，真实分享创建必须保持关闭。
